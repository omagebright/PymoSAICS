# MOSAICS parameter decks and hybrid topology files

Force-field parameter databases for parmbsc0 and parmbsc1, plus the
hybrid CHARMM RTFs used for DNA/RNA heteroduplex topologies.

## Layout

```
params/
  db_bsc1/                                       parmbsc1 parameter deck
    mosaics_amber99-bsc1.bond                    bond force constants and r0
    mosaics_amber99-bsc1.bend                    bend force constants and theta0
    mosaics_amber99-bsc1.tors_and_impr           proper + improper torsions (cosine power-series form)
    mosaics_amber99-bsc1.onfo                    1-4 (1-fourier) non-bonded pair list
    mosaics_amber99-bsc1.vdw                     Lennard-Jones and Coulomb parameters
  db_bs0/                                        parmbsc0 parameter deck (same structure)
  db_ol21_ol3/                                  OpenMM-derived OL21/OL3 deck, standard/internalized profile
  db_ol15_ol3/                                  OpenMM-derived OL15/OL3 deck, standard/internalized profile
  db_ol24_ol3/                                  AMBER-prmtop-derived OL24/OL3 deck, standard/internalized profile
  db_ol21_ol3_terminal/                         OpenMM-derived OL21/OL3 deck, true-terminal profile
  db_ol15_ol3_terminal/                         OpenMM-derived OL15/OL3 deck, true-terminal profile
  db_ol24_ol3_terminal/                         AMBER-prmtop-derived OL24/OL3 deck, true-terminal profile
  db_charmm36/                                  excluded artifact, not part of this release scope
  top_all99-bsc1_prot_hybrid_chidef.rtf          assembled hybrid RTF (what MOSAICS reads)
  top_all99-bs0_prot_hybrid_chidef.rtf           assembled hybrid RTF (parmbsc0 variant)
  top_openmm-ol21_ol3_hybrid_chidef.rtf          hybrid RTF patched with OpenMM OL21/OL3 atom types and charges
  top_openmm-ol15_ol3_hybrid_chidef.rtf          hybrid RTF patched with OpenMM OL15/OL3 atom types and charges
  top_amber-ol24_ol3_hybrid_chidef.rtf           hybrid RTF patched with official AMBER OL24/OL3 atom types and charges
  top_openmm-ol21_ol3_terminal_hybrid_chidef.rtf true-terminal OL21/OL3 hybrid RTF
  top_openmm-ol15_ol3_terminal_hybrid_chidef.rtf true-terminal OL15/OL3 hybrid RTF
  top_amber-ol24_ol3_terminal_hybrid_chidef.rtf  true-terminal OL24/OL3 hybrid RTF
  forcefield_panel.tsv                           implementation status for the approved panel
  rtf_sources/                                   modular source files for the bsc1 RTF
    top_all99-bsc1_chidef_core.rtf               internal residues only (ADE/URA/CYT/GUA + ADD/THD/CYD/GUD)
    top_all99-bsc1_chidef_terminals.rtf          16 terminal templates (AD5/AD3/.../UR5/UR3)
```

## Modular RTF source

MOSAICS' RTF parser reads a single file. To keep the source editable
without one monolithic 1400-line file, the bsc1 hybrid RTF is split
into two source files under `rtf_sources/`. The assembled
`top_all99-bsc1_prot_hybrid_chidef.rtf` in this directory is what
MOSAICS actually reads; it is generated by concatenating the source
files using `tools/assemble_rtf.py`. Edit the source files when you
want to change anything, then re-run the assembler.

The bs0 RTF currently has no terminal-variant templates (16 more
templates would need extracting from `RNA.lib` + `parmbsc0.lib`), so
it stays as a single unsplit file for now.

A future MOSAICS code change could add a CHARMM-style `STREAM`
directive to the RTF parser, which would let the loader concatenate
files at parse time and eliminate the assembler step entirely.

## Fixes baked in

The decks and RTFs in this directory carry the following corrections
relative to the May 2026 starting state:

- **Bug D** -- `*.tors_and_impr`: impropers expressed in cosine
  power-series form. The earlier shipped decks had a factor-of-2
  harmonic-form artefact from the converter; that is removed.
- **Bug E** -- `*.tors_and_impr`: Perl converter radians / IDIVF
  coefficient fix. Earlier decks had a units mistake in the proper-
  torsion coefficient transcription; that is corrected.
- **Bug G** -- `*.vdw`: Coulomb cutoff `max_dist{10}` raised to
  `max_dist{100}` so the non-bonded list reaches all 1-4 pairs in
  the typical nucleic-acid backbone geometry.
- **Bug Y** -- `db_bsc1/*.bend`: missing `CE-CT-OH` RNA bend added.
  Exposed by the parmbsc1 RNA chi correction (which renames C3'
  to CE in pyrimidines).
- **Bug Z** -- `db_bsc1/*.tors_and_impr`: missing C1 improper twins
  of CK impropers added.
- **Bug H** -- `top_all99-bsc1_prot_hybrid_chidef.rtf`: RNA C3'
  reverted from CE back to CT. The over-applied parmbsc1 rename
  was breaking RNA torsion lookups.
- **RTF terminal-variant extension** -- 16 new residue templates
  (4 hybrid DNA + 8 3'-OH terminal + 8 5'-OH terminal). Extracted from
  AMBER `parmBSC1.lib` and `RNA_CI.lib` with the appropriate
  hybrid atom-name aliases. Lets MOSAICS handle arbitrary 5'-OH and
  3'-OH terminated nucleotides without external preprocessing.
  Source lives in `rtf_sources/top_all99-bsc1_chidef_terminals.rtf`
  and is concatenated into the assembled canonical RTF by the
  assembler script.

## How decks are generated

`tools/convert_charmm_tors.pl` is the Perl converter that produces
the `*.tors_and_impr` deck from AMBER's CHARMM-style frcmod output.
See `tools/README.md`.

The 2026-05-27 expanded panel decks are generated by
`tools/openmm_xml_to_mosaics.py` for OpenMM XML sources and by
`tools/amber_prmtop_to_mosaics.py` for official AMBER source packages
without local OpenMM XML files. Current generated artifacts are:

- `OL21/OL3`: `db_ol21_ol3/` plus
  `top_openmm-ol21_ol3_hybrid_chidef.rtf`.
- `OL15/OL3`: `db_ol15_ol3/` plus
  `top_openmm-ol15_ol3_hybrid_chidef.rtf`.
- `OL24/OL3`: `db_ol24_ol3/` plus
  `top_amber-ol24_ol3_hybrid_chidef.rtf`.
- `OL21/OL3 terminal`: `db_ol21_ol3_terminal/` plus
  `top_openmm-ol21_ol3_terminal_hybrid_chidef.rtf`.
- `OL15/OL3 terminal`: `db_ol15_ol3_terminal/` plus
  `top_openmm-ol15_ol3_terminal_hybrid_chidef.rtf`.
- `OL24/OL3 terminal`: `db_ol24_ol3_terminal/` plus
  `top_amber-ol24_ol3_terminal_hybrid_chidef.rtf`.

`CHARMM36+sHBfix` is out of scope for this project. The local
`db_charmm36/` directory is not a release target and should not be
advertised as a usable MOSAICS deck.

These directories are implementation artifacts, not final production
approvals. The matched `1EFS` AMBER/OpenMM/MOSAICS single-point gate
passes for `OL21/OL3`, `OL15/OL3`, and `OL24/OL3`. Reduced DNA/RNA
single-residue and dimer systems extracted from `1EFS` also pass for all
three decks. True terminal pure-DNA and pure-RNA full-system checks pass
for the terminal profiles. The remaining pre-release work is usability hygiene: profile
selection documentation, public conversion instructions, fresh-clone
smoke tests, and provenance notes.
`tools/audit_mosaics_deck_coverage.py` is the static RTF-to-deck
coverage gate; it passes for `OL21/OL3`, `OL15/OL3`, and `OL24/OL3`.

## Supported public-use scope

The validated AMBER-family release scope is canonical DNA, canonical
RNA, and canonical DNA/RNA hybrid systems with the residue and atom
names represented in the hybrid RTF files above. Bulges and mismatches
are supported as sequence/topology variants if they use canonical bases
and the same atom naming.

Use the standard profile for matched `1EFS`-style internalized
heteroduplex systems. Use the terminal profile for systems with explicit
5-prime and 3-prime terminal residues.

Unsupported in this release: modified bases, ligands, protein-ligand
complexes, explicit solvent as mobile particles, mobile ions, polarizable
force fields, and `CHARMM36+sHBfix`.
